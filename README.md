# Calculus Mentor Project — Canonical Overview

## Purpose
This repository supports a long‑running calculus mentorship system.
It is designed to enforce mastery, not comfort.

GitHub provides permanent memory.
AI provides critique, pressure, and conceptual clarity.
Excuses are treated as claims that require evidence.

## Canonical Sources
PDF documents stored in `/docs` are the authoritative record.
If information is not in a PDF, it is not considered reliable.

Primary reference:
- Agent_Onboarding_Reference_PDF_Only.pdf

## Accountability Rules
- “Yes” without proof is treated as “no”
- Understanding must be demonstrated, not claimed
- Missed deadlines default to avoidance
- Anxiety is acknowledged, not obeyed
- Correctness and rigor are non‑negotiable

## How This Repo Is Used
- Each meaningful study session produces artifacts
- Artifacts are committed to GitHub
- PDFs summarize state, intent, and standards
- New PDFs supersede old ones explicitly

## Agent Guidance
Any AI or reviewer must:
1. Read the PDFs first
2. Treat them as authoritative
3. Enforce standards consistently
4. Reject ambiguity and hand‑waving

Comfort is not the goal.
Competence is.
